Your First Assignment: Work with the Starter Code

Now that you've got the code, you're ready to become comfortable with it. Here we will walk you through the starter code and making some small changes to it. Make sure the project you downloaded and set up for this course is open in eclipse.

As you saw in Mia's demo video, all of the functionality you will implement in this course will be integrated into a text editor application. You can already run this application, but it won't do much.

Run the text editor by running the MainApp class inside the application package. You can do this by expanding the application package in the package explorer and then selecting the MainApp.java file and running it.

You will see a text editor window open. You can type in the text window and load text using the "Load" button. None of the other buttons work yet (you can click them, but they don't do anything). But they will soon...

3. Orient yourself to the starter code

Open the starter code for this assignment by expanding the document package in the Package Explorer window. There you will see several files, but the two that are relevant to your first real programming assignment (next week) are Document.java and BasicDocument.java. Open these by double-clicking on them.

Notice that Document is an abstract class. Next week you will add code to BasicDocument to implement the abstract methods in the Document class. But for this week we'll just play with the Document class.

4. Run the MainApp and type some text in the box, then click the "Flesch Index" button. Notice that the number in the box at the bottom says 0.0. This is because the method getFleschScore() in the Document class always returns 0.0 right now. What's happening is that when you type text in the box and then click the Flesch Index button, the application is creating a new Document object (actually an EfficientDocument object, since Document is an abstract class) and calling the getFleschScore() method to determine the value to put in the box.

Change the value returned by getFleschScore() in Document.java then relaunch the MainApp and see what happens when you type some text and click the Flesch Index button. You should see the number that you returned appear!

5. Modify the getFleschScore so that it returns the number of characters of the text in the box . Hint: the text in the box is stored in a member variable named "text" in the Document class, and you can figure out the number of characters in a String using the "length()" function which is defined for the String class. Then relaunch the MainApp and notice that when you click Flesch Index, the number in the box is the number of characters in the text box.

If you like you can continue to play around with the code, and explore the other classes we provide. Their function will be revealed as you work through the course. At this point you're ready to take the very short quiz that follows, and then move on to next week's work!

Q: I'm getting a "Grader Failed" message when I submit my code. What's wrong? Is this a problem with my code or with your grader?

A: At this point our graders have been tested extensively, so chances are it's something in your code. The most common reason for getting the "grader failed" error is that your code does something very unexpected, such as incorrect return types, infinite loops, thrown exceptions, or other similar errors. So if you are getting this error check your code carefully and run the grader locally to make sure everything is exactly as it should be. If you still cannot catch your mistake, try posting on the forum to see if anyone else has received a similar message. Finally, if that doesn't fix the problem, go ahead and submit a bug report by clicking the flag button on the bottom of the assignment page and we will look into it further. But again, chances are it's something wrong in your code and you'll solve the problem fastest by trying to get to the bottom of things yourself.

Q: My fleschScore method returns the wrong value/I am failing the tests for Part 2.

A: The number one reason that people get the wrong value when they calculate the Flesch score is that they fail to cast the ints returned by getNumWords, getNumSentences and getNumSyllables to doubles before they divide them. This will cause Java to use integer division and truncate numbers inappropriately.

If this is not your problem, make sure each of your getNumWords, getNumSentences and getNumSyllables is returning the correct value, and that you don't have a typo in the constants used in the formula. Finally, check the order of operations, and add parentheses if necessary.
